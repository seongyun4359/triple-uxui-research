## 4. 핵심 내용 요약

### 4.1. ‘트리플’ 앱 선정 이유

- 코로나 이후 **여행 수요의 폭발적인 증가**와 함께,  
  여행 준비를 한 번에 해결할 수 있는 **올인원 여행 가이드 앱**의 중요성이 커졌다.  
- 트리플은 이미 **여행 일정·예약·가계부·체크리스트·리뷰**를 아우르는 강력한 앱이지만,  
  실제 사용 과정에서 **가시성, 편의성, 기능 부족, 실시간 정보, 일관성** 측면의 한계를 보여 주었다.  

### 4.2. 트리플의 주요 기능

- 여행지(관광지, 숙소, 맛집) 추천 및 **항공/숙소/티켓 예약**  
- 여행 일정 등록 및 **지도 기반 일정 확인**  
- **가계부, 체크리스트** 기능으로 여행 관련 기록 통합  
- **여행지 날씨, 지도, 교통, 환율, 번역** 등 부가 정보 제공  
- **리뷰·평점 공유**를 통한 사용자 경험 교환

### 4.3. 주요 UX 문제와 개선 방향

- **홈 화면 UX**
  - 국내/해외 탭 구분이 초기 진입 단계에서 불필요하게 복잡함  
  - 여행 계획 vs 여행지 추천 **카테고리 구분이 모호**  
  - 등록된 여행 일정 카드의 **크기·가시성이 낮아 인지 어려움**

- **일정 계획 화면**
  - 구글 맵과 트리플 간 **장소 이름·정보 불일치**  
  - **영업 여부, 이동 시간, 동선 기반 재배치** 기능 부족  
  - 다수 일정 **일괄 수정 불가**, 하루 하나의 일정만 등록 가능  
  - 가계부·체크리스트 버튼 위치가 비직관적

- **개선 방향 (요약)**
  - 정보 구조 재설계: **홈·여행 매거진·예약·일정·마이페이지·커뮤니티** 중심 구조  
  - **생성형 AI 기반 여행 일정 추천**, 동선 최적화 및 이동 수단/소요시간 가시화  
  - **하루 복수 일정·대안 일정** 추가, 전체 일정 일괄 편집  
  - **오프라인 일정 확인**, 가계부/체크리스트 UI 재배치

## 5. 전통적 UX/UI 프로세스 (A안)

- **Competitive Review**: 위시빈, 스투비플래너, 트립어드바이저, 핫츠고플랜 등 비교  
  - 장점: 깔끔한 UI, 일정 공유, 최적 동선 기능, 다양한 투어·체험 상품 등  
  - 단점: 예약 노출 부족, 가격·할인 정보 부족, 광고 편향, 오류 빈도 등  
- **Persona**:  
  - 디지털 기록을 선호하는 잦은 여행자,  
  - 첫 해외여행을 준비하는 초보 여행자 설정 후 니즈 분석  
- **Sitemap · Flowchart · Wireframe**:  
  - **계층 구조·사용 플로우**를 명확히 정의하여 앱 구조 개선  
- **High-Fidelity Prototype (Figma)**:  
  - **네비게이션바·지도·일정 카드·리뷰 영역**의 일관성과 가독성 강화  
- **Usability Test (SUS)**:  
  - 20대 사용자 5명, 평균 **85점**  
  - 개선 포인트: 사용 방법 단순화, 여행지 정보 확장, 일정 화면의 지도 크기·정보 밀도 조정

## 6. AI 기반 프로세스 (B안)

- **Persona 생성**: ChatGPT + QoQo.AI + Playground  
  - 키워드 기반 상세 프로필, 성격 유형(INFJ), 직업·취미·여행 스타일 자동 생성  
- **Wireframe Designer (Figma + AI)**  
  - 페이지별 필수 요소·버튼·아이콘 정보를 인풋으로 제공 후 와이어프레임 자동 생성  
  - AI 결과물을 바탕으로 **한국어 번역·레거시 구조 맞춤 수정** 진행  
- **Prototyping with ChatGPT**  
  - 브랜딩, 레이아웃, 모션, 접근성, 네비게이션 명명, 오류 예방 패턴 등 제안 활용  
- **Heuristic Evaluation (AI)**  
  - Nielsen 10 Heuristics 기준 자동 평가  
  - 시스템 상태 가시성, 실제 세계와의 매칭, 오류 예방, 인지 부하 감소 등 세부 코멘트 획득

## 7. A안 vs B안 비교 인사이트

- **Persona**
  - A안: 현실적인 감정·맥락이 풍부, 실제 데이터·인터뷰 기반  
  - B안: 배경·성격·필요 조건을 빠르게 생성하지만, 감정·맥락·다양성은 제한적  

- **Wireframe/Prototype**
  - A안:  
    - 장점: 디자이너 의도와 구조를 자유롭게 반영, 맥락 이해에 유리  
    - 단점: 제작·수정에 시간 많이 소요, 객관성 확보 어려움  
  - B안:  
    - 장점: 초기 레이아웃·구조 설계 속도 매우 빠름, 수정·실험이 쉬움  
    - 단점: 복잡한 기능 이해 부족, 독창성·브랜딩 측면에서 추가 보정 필요  

- **Usability Test**
  - A안: 사람 중심의 깊이 있는 해석, 하지만 시간과 인력 소모 큼  
  - B안: 빠른 피드백과 일관된 기준 제공, 그러나 실제 사용자 경험과 괴리가 있을 수 있음  

**결론적으로**,  
전통적인 UX 프로세스(A안)는 **깊이·맥락·현실성**에서 강점을,  
AI 기반 프로세스(B안)는 **속도·다양한 아이디어·반복 실험 용이성**에서 강점을 보였다.  
두 접근을 **혼합(Hybrid)** 했을 때 가장 효율적이고 풍부한 UX/UI 개선이 가능하다는 인사이트를 얻었다.
